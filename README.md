# News-to-Trade Relevance Scorer

Демо-сервис для трейд-анализа коротких финансовых текстов: заголовков новостей, твитов и фрагментов пресс-релизов.

Приложение предсказывает:
- настроение — `sentiment` (`bullish` / `bearish` / `neutral`);
- влияние — `actionability` (`actionable` / `non_actionable`);
- тип события — `event_type`;
- торговый горизонт — `horizon` (`HFT`, `MFT`, etc.);
- степень уверенности в предсказании;
- короткое текстовое объяснение.

**[Ссылка на сайт сервиса](https://savidan-news-scorer.streamlit.app)**

## Как это устроено

Сервис использует три отдельно дообученные модели на базе FinBERT для следующих задач соответственно:
- sentiment;
- actionability;
- event type.

Для деплоя используется `ONNX Runtime` и `Streamlit`. Поле `horizon` вычисляется эвристически из предсказанных `actionability` и `event_type`.

## Проведённые эксперименты

В рамках проекта были рассмотрены и дообучены несколько отдельных моделей на базе FinBERT:
- `sentiment`-модель: мультиклассовая классификация (настроение);
- `actionability`-модель: бинарная классификация (влияние);
- `event type`-модель: многоклассовая классификация (тип события).

Для `event type` также тестировалась двухфазовая схема обучения:
- сначала на расширенном тренировочном датасете;
- затем дополнительное дообучение на избранной подвыборке.

Для `actionability` и исследовались и дополнительно рассматривались:
- обычный полный fine-tuning;
- более мягкий fine-tuning с уменьшенным learning rate;
- вариант с заморозкой энкодера;
- PEFT (LoRA) с варьированием гиперпараметров.

По итогам экспериментов, лучшими оказались:
- `sentiment` — полный fine-tuning FinBERT;
- `actionability` — полный fine-tuning FinBERT;
- `event type` — LoRA на FinBERT.

**Исходники экспериментов** расположены в папках `models/exp_<...>`.

## Работа с данными

Для обучения использовались следующие датасеты:
- **Financial PhraseBank** для задачи `sentiment`;
- **FiQA** как основное ядро для задач `actionability` и `event type`;
- внешний набор (с Kaggle) финансовых headline'ов для раздувания размера тренировочной выборки.

Что было сделано с данными:
- нормализация текстов и схемы полей;
- эвристическая разметка с использованиеем регулярных выражений и ключевых слов;
- ручная валидация датасетов;
- объединение датасетов из разных источников разметок;
- выделение для тестирования только тех семплов, которые были вручную провалидированы.

## Метрики и результаты

Модели для задачи `actionability` тестировались по метрикам `Accuracy` и `F1-score`. Для задач `event type` и `sentiment` использовались `macro`-аналоги соответствующих метрик.

Ниже таблица с детальным описанием достигнутых метрик на тестовом датасете по каждой из задач:

| Задача | Accuracy | F1-score |
| --- | --- | --- |
| `Sentiment` | $0.9594$ | $0.9539$ |
| `Actionability` | $0.8245$ | $0.8171$ |
| `Event type` | $0.8198$ | $0.8112$ |
